#### Integration of XP Practicies to enhance general group work:

* Pair Programming: More experienced SW developers (eg Maurin & Emilio) will review the code written by other (less experienced) team members.
* Automated Testing: Tests should be implemented before the actual code. This helps to ensure that the code meets all needed requirements.
* Continuous Integration: Frequently merging feature branches.
* Simple Design: The focus will be on creating a simple system, not an overcomplicated one.
* Workspace Information: The whole project will be splitted in different tasks @ git
* Team Meeting: There will be weekls talks with all group members where we can talk about the current issues & tasks.
* Individual Meetings: Some pairs can meet all the time on an individual base (fex. Codereviews or Merge Requests)
* Team Communication: Most of the communication is beeing made in person or with the help of (mostly) WhatsApp or Teams Group channels.
