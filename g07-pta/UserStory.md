
## USER STORY

### Was ist eine User Story?

Kurzbeschreibung wer was warum will - ganz einfach, für jeden verständlich ausgedrüvckt.

Eine User Story beschreibt die Anwendersicht auf einfachster abstrahierter Ebene. Es soll mit worten die Funktion
und Nutzen des Projektes aufgezeigt werden. Das ganze ist in Alltagssprache formuliert und somit kein techn. Dokument.

### Aufbau einer User Story:
    
Hier sind 3 Zentrale Punkte wichtig:\
    - **Rolle der Benutzer** Für wen möchten wir unser Produkt entwicklen? Wer ist unser typischer User?\
    - **Absicht der Benutzer** Welchen Zweck möchte der Kunde erreichen? Was will der User machen?\
    - **Nutzen der Benutzer** Was will der User genau erreichen? Was für ein Mehrwert bringt das ihm?

### INVEST Kriterien für eine User Story:

I: Independent -> Die User Story soll unabhängig von anderen sein.\
N: Negotiable -> Die User Story soll Handlungsspielraum erlauben.\
V: Valuable -> Die User Story soll einen Erkennbaren Nutzen für den Kunden aufzeigen.\
E: Estimable -> Die User Story soll nicht zuviel Zeit beanspruchen.\
S: Small -> Die User Story soll klein sein & schnell gemacht sein. (in Sprint Meeeting!)\
T: Testable -> Die User Story soll überprüfbar sein.
